Tables

Guiding Principles

Tables show structured data and their purpose is to make that data readable, scan-able and easily comparable. The standardization of data table structure provides a framework of functionality that can be included or excluded from a layout as needed. The general direction is that we only expose the table elements necessary for the required task and no more, but because we are using a common framework there will be consistency in how the elements are used.
Tables are often much more than a plain placeholder for endless data. They also can be interactive elements that can perform specific actions. Actions are usually performed on a single row but they can be also performed on multiple rows. Actions can be shown inline, they can be shown as a menu, or as a hyperlink on the data itself. The complexity of the configuration form that produces the data to display influences how you can edit that data from the table. All tables have a max-width of 1300px, unless otherwise specified.
