Buttons

Guiding Principles

Buttons are typically used to execute an action on a page, navigate to another page in a wizard style flow, open a new modal, or close the current modal. Don't use ellipses (...) in the button label unless it opens a new modal. Do not use arrows in buttons unless it is in the specifications. Consider using link tags for "Cancel" and "Close" actions. JIRA has good examples of this. Be careful not to let technical jargon slip into buttons.
"Save" is generally well understood for saving changes, but "Insert" in the context of creating content typically is not. I think for the most part we are pretty good about using understandable terms such as "Update", "Save", "Create, etc... The ordering of buttons weather in an active window, framed popup, or modal follows the structure below "\ close \ primary \ success \ danger".

Large All Caps

Used for CTA actions such as 'CREATE NEW' 'SIGN IN' ect.

Large - Camel Case

Used for navigation, and any other system action where white space permits a larger button size. Main content, Modal footer, ect. 'Next' 'Back' 'Cancel' ...

Small - Camel Case

Used for navigation, and any other system action where white space dictates a smaller button size. Main content only 'More Options' Any button or dropdown located inside a data table.
