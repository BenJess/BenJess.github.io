Navigation

Guiding Principles

On desktop devices hamburger menus, such as the example shown in this guide, should load fully expanded for new users. Its collapsed or expanded state should be remembered between page views. On tablet devices it always loads collapsed but can be expanded, but it will not stay expanded on subsequent page views. The relevant navigation item should be "selected" based on what tool you are in. (ie if you are on any of the Email Marketing pages "Email Marketing" should be highlighted). When you hover over an item it should become highlighted.
The Progress indicator is a step list. It guides the user through a number of steps in order to complete a specified process. It is implimented in such modules as Email and Event creation, and commonly reffered to as a "wizard". The Progress Inidactor is not to be confused with a breadcrumb list, which is a way of enhancing navigation by revealing a userâ€™s current location. Global breadcrumbs will remain the same to provide an easy transistion for existing users into the new redesign.

Progress Indicator

The wizard progress indicator may only be used in full-screen instances of a module, and never in a multi-step modal. Wizard progress indicators may only be used when the series of steps needed to complete a task ENDS in a commit or save function. If a commit function is available mid-steps, or bouncing back and forth between pages or steps is allowed, progress is indicated by Breadcrumb trail. Breadcrumb trails are also the only acceptable means of step progression in multi-step modals, and popup browser windows.
